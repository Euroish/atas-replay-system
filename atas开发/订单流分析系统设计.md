# 历史草案说明

本文件保留项目早期需求整理和方案探索，作用是记录“问题是如何被提出的”，而不是定义“现在项目是什么”。当前正式口径一律以 `atas开发/project.md` 为准。

## 本文件的定位

- 保留早期想法、问题表述、备选路线和提示词素材。
- 帮助回溯为什么会有 `Heatmap`、`Footprint`、多窗口、订单簿重建等核心方向。
- 不直接承担需求冻结、架构定稿、接口定义和验收标准职责。

## 使用规则

1. 如果这里只是历史想法，而 `project.md` 没采纳，则视为未生效方案。
2. 如果这里的某条内容后来被确认，需要改写进 `project.md` 对应章节，而不是只留在本文件。
3. 实施开发、拆任务、做验收时，不引用本文件作为唯一依据。

## 这份草案贡献过的核心方向

- 系统必须以订单流回放为核心，而不是普通 K 线软件。
- 图形模式以 `Heatmap` 和 `Footprint` 为主。
- 数据正确性优先于 UI 复刻。
- 多股票、多日期、多窗口和基础指标是目标能力的一部分。
- `DOM`、挂单撤单可视化、Delta、Volume Profile、POC 都需要纳入正式设计。

## 当前推荐阅读方式

1. 先读 `atas开发/project.md` 获取当前完整口径。
2. 仅当需要回看需求来源、早期取舍或提示词素材时，再读本文件。
